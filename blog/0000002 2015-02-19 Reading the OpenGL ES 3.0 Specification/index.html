Chapter 2: OpenGL ES Operation

OpenGL is a language for producing images in frame buffers. It does this via drawing primitives within certain modes and by using shader programs.

Definitions:

Primitive: A point, line segment or triangle. Consists of one or more vertices.

Vertex: A point, an endpoint of an edge or a corner of a polygon here two edges meet.

Commands are always processed in order but are asynchronous. So for example pixel read operations return the state of the data as one would expect, the data as it is after all previous OpenGL commands have been executed.

Data passed to OpenGL commands are interpreted right then, not when the asynchronous OpenGL execution pipeline reaches that stage. This includes pointers.
